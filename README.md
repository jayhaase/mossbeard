# Mossbeard the Wanderer

Website for Mossbeard the Wanderer — a Renaissance faire character and keeper of quiet trails and old tales.

**Live site:** [mossbeard-the-wanderer.com](https://mossbeard-the-wanderer.com)  
**Facebook:** [Mossbeard the Wanderer](https://www.facebook.com/people/Mossbeard-the-Wanderer/61580016592890/)

---

## Deploying

This site uses [Jekyll](https://jekyllrb.com/), which GitHub Pages builds automatically.

**No local build step is required.** Just push your changes to GitHub — the Actions workflow builds and deploys the site automatically.

To enable GitHub Pages for the first time:
1. Push the repo to GitHub
2. Go to **Settings → Pages**
3. Set Source to **GitHub Actions** (not "Deploy from a branch")
4. The next push will trigger a build — the live URL appears at the top of that page once it completes

Build status is visible under the **Actions** tab in the repo.

CSS and JS cache-busting is automatic via the `_plugins/cache_bust.rb` plugin (MD5 content hash appended when files change).

## Editing content

All content lives in plain text files. You do not need to touch `index.html` or `_layouts/` for routine updates.

### Site copy, hero, and links — `_data/site.yml`

Edit hero text, navigation labels, section titles/intros, Facebook URL, and the Connect CTA here.

### Faire appearances — `_data/appearances.yml`

Add, remove, or reorder blocks to update the *Find Mossbeard* section.

```yaml
- name:        My New Faire
  location:    City, State
  date:        October 2026
  ends_on:     2026-10-31   # last day to show; card drops off on the next site build
  upcoming:    true        # shows the "Upcoming" badge when true
  description: A wonderful faire in the autumn woods.
  url:         https://example.com
  link_text:   Visit the Faire
```

Past appearances are hidden automatically when the site rebuilds (each push to GitHub). Set `ends_on` to the faire’s last day.

### Gallery photos — `_data/gallery.yml`

1. Add a **kebab-case** JPEG to `photos/` (e.g. `photos/my-new-photo.jpg`).
2. Run `./scripts/optimize-photos.sh` to create WebP and thumbnail variants.
3. Add an entry (copy fields from an existing row; get `width` / `height` with Preview or `sips -g pixelWidth -g pixelHeight photos/my-new-photo.jpg`):

```yaml
- src:        photos/my-new-photo.jpg
  webp:       photos/my-new-photo.webp
  thumb:      photos/thumbs/my-new-photo.jpg
  thumb_webp: photos/thumbs/my-new-photo.webp
  width:      1280
  height:     1920
  alt:        A full description for screen readers
  label:      Short label shown in the lightbox
```

Reorder entries to change the gallery order. Update `hero.image` in `_data/site.yml` if you change the hero photo.

### Magical relics — `_data/relics.yml`

Each relic uses an SVG icon or a photo:

```yaml
# Icon relic
- name:        The New Artifact
  svg:         icons/compass.svg
  description: What this artifact does.

# Photo relic (omit svg)
- name:        The Visual Artifact
  photo:       photos/artifact.jpg
  photo_alt:   Description of the photo
  description: What this artifact does.
```

### Prose sections — `_includes/content/`

| File | What it controls |
|------|-----------------|
| `wanderer-lore.md` | The origin story paragraphs in *The Wanderer* |
| `forgotten-words.md` | The *Forgotten Words* prose (use `> text` for the mission blockquote) |

Standard Markdown formatting works in both files.

### Site title & description — `_config.yml`

Update `title` and `description` at the top of `_config.yml`.

## Previewing locally (optional)

You only need this if you want to see changes before pushing to GitHub.

**Prerequisites:** Ruby 3.3+ and Bundler:

```bash
gem install bundler
bundle install
```

**Start the local server:**

```bash
bundle exec jekyll serve
```

Then open [http://localhost:4000](http://localhost:4000). The site reloads automatically when you save files.

## Project structure

```
_config.yml              — site settings, Jekyll plugins
_data/
  site.yml               — hero, nav, section copy, Facebook URL
  appearances.yml        — faire dates and locations
  relics.yml             — magical artifacts
  gallery.yml            — gallery photos (paths + dimensions)
_includes/
  sections/              — one HTML partial per page section
  content/
    wanderer-lore.md     — Wanderer section body
    forgotten-words.md   — Forgotten Words section prose
_layouts/
  default.html           — HTML shell (head, footer, scripts)
index.html               — assembles section includes
css/style.css            — all styles
js/main.js               — gallery lightbox, mobile nav, sticky nav
photos/                  — full-size JPEGs + WebP
photos/thumbs/           — grid thumbnails (generated by script)
scripts/
  optimize-photos.sh     — resize + WebP + thumbs for new photos
docs/                    — source lore documents (not published)
```
